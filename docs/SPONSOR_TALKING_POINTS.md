# Sponsor Talking Points

## Terac

We built a custom annotation workflow for accessibility-grade sports descriptions. Labelers compare baseline commentary with MatchVision output and score ball location, direction of attack, key-event coverage, concision, hallucination, and helpfulness. The label data powers measurable before/after improvement.

## Deepgram

Voice is central because blind and low-vision fans should not need a visual UI. MatchVision supports voice questions and spoken descriptions. The intended production path uses Deepgram STT and TTS / voice agent APIs.

## Anthropic

Claude is used to transform structured match state into concise, accessible descriptions and answer follow-up questions. The project has a meaningful inclusion impact: making mainstream sports video accessible to fans excluded by visual-first broadcasts.

## Redis

Redis is the real-time memory layer for match state, recent events, and user preferences such as brief, tactical, beginner, or emotional mode. It enables follow-up questions like “what changed?” or “who has space now?”

## Arize

Arize fits because we evaluate generated descriptions, not just demo them. We track helpfulness, event coverage, spatial coverage, and hallucination rate before and after human labels.

## Ddoski's World

MatchVision is a social impact accessibility project. It uses AI to expand participation in a global cultural experience for blind and low-vision fans.
